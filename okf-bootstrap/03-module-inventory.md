# Module Inventory

## Inventory Rules

- Module means a coherent runtime capability, not necessarily one Rust module.
- Business rules are included only when directly evidenced by code, migrations, tests, or current documents.
- Documentation status describes whether a consolidated, current module document exists.
- Twenty-two primary modules were identified.

## Module Boundary Findings

- Backend module names are technical rather than strict domain packages.
- routes/marketplace.rs spans several Marketplace modules and is the largest backend route file.
- PagesPage.tsx, MarketplacePage.tsx, and OrganizationPage.tsx span multiple UI responsibilities.
- models is not a complete domain-model layer; route-local DTOs and SQLx rows are more authoritative.
- The future OKF should document capability modules while linking their cross-cutting route/service files.
