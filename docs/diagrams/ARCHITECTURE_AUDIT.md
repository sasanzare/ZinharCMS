## RLS Matrix

Evidence paths: `backend/migrations/0008_v2_phase_one_organizations.sql`, `backend/migrations/0009_v2_phase_three_rls.sql`, `backend/migrations/0010_v2_phase_five_billing_quota.sql`, `backend/migrations/0011_v2_phase_six_stripe_billing.sql`, `backend/migrations/0012_v2_phase_seven_saas_ops.sql`, `backend/migrations/0014_v2_phase_nine_beta_release.sql`, `backend/migrations/0015_v3_phase_one_marketplace_foundation.sql`, `backend/src/middleware/tenant.rs`, and `backend/src/services/rls.rs`.

Runtime context: tenant routes require `X-Organization-Id`, an access token, an active organization, and an active `organization_members` row. RLS-scoped queries call `rls::tenant_connection`, `rls::begin_tenant_transaction`, or `rls::organization_connection`, which set `zinhar.organization_id`, `zinhar.user_id`, and `zinhar.rls_bypass=false`. The only explicit bypass helper found is `rls::begin_bypass_transaction`, which sets `zinhar.rls_bypass=true`; a `super_admin` role does not automatically bypass tenant middleware.
